## Recent Changes (2026-06-28)

### 1. FIFA R32 Bracket Structure
**File:** `model/monte-carlo.js` — `buildKOBracket()`

The bracket array was completely rewritten to match the official FIFA schedule:
- Position [0]=M73 (R(A) vs R(B)), [1]=M74 (W(C) vs R(F)), etc.
- Previously had wrong match order and wrong team assignments

**Key change:** The bracket now returns 32 entries in FIFA schedule order (M73→M88), not in a derived order.

### 2. Third-Place Assignment
**File:** `model/monte-carlo.js` — `buildKOBracket()`

Uses backtracking with slot constraints to assign 8 third-place teams to 8 winner slots:
- Each slot has `eligible` groups (excluding self and R16 opponent)
- Adjacent match's third-place team is also excluded
- Greedy fallback for edge cases

**Known limitation:** When the simulation's predicted third-place teams differ from actual FIFA results, the bracket may show different matchups. This is inherent to the model's prediction accuracy.

### 3. R16 Cross-Pairing
**File:** `compute-server.js` — `runKORound()`

R16 pairings use bracket indices (not sequential):
```
M89: w1[1] vs w1[4]   // W(M74) vs W(M77)
M90: w1[0] vs w1[2]   // W(M73) vs W(M75)
M91: w1[3] vs w1[5]   // W(M76) vs W(M78)
M92: w1[6] vs w1[7]   // W(M79) vs W(M80)
M93: w1[10] vs w1[11] // W(M83) vs W(M84)
M94: w1[8] vs w1[9]   // W(M81) vs W(M82)
M95: w1[13] vs w1[15] // W(M86) vs W(M88)
M96: w1[12] vs w1[14] // W(M85) vs W(M87)
```

### 4. Beijing Time Display
**File:** `index.html` — `KO_INFOS`, `R32_INFO`, `R16_INFO`, etc.

All knockout match times converted from ET to Beijing time (ET+12h). Example:
- M76 Netherlands vs Morocco: 9pm ET Jun 29 → 9:00 Beijing Jun 30

### 5. Knockout Match Predictions
**File:** `compute-server.js` — `detScore()`

Changed from random Poisson sampling to deterministic expected goals:
```javascript
function detScore(home, away, lh, la) {
  const ga90 = Math.round(lh), gb90 = Math.round(la);
  if (ga90 !== gb90) return { ga90, gb90, ga: ga90, gb: gb90, method: "90'", lh, la };
  // Draw → Elo-based penalty winner
  ...
}
```

Returns `lh` and `la` (expected goals) for xG display in the UI.

### 6. Async Race Conditions Fixed
**File:** `index.html` — multiple functions

Fixed fire-and-forget async calls in:
- `refreshData()` — now `await reevaluateResults(); await runSimulation()`
- `addResult()` — now `async function` with `await`
- `importResults()` — now `async function` with `await`
- `applyCustomOdds()` — now `async function` with `await`

### 7. Frontend Stub Cleanup
**File:** `index.html`

Removed calls to frontend stub functions that returned hardcoded values:
- `WC26.getBlendedProbs()` — was returning `{win:0.33, draw:0.34, loss:0.33}`
- `WC26.getFormAdjustedLambdas()` — was returning `[1.2, 1.0]`
- `WC26.predictOutcome()` — was using stub probs

Now all probability computation goes through the server.

### 8. Auto-Fetch Before Simulation
**File:** `index.html` — init flow

Changed page load order:
1. Auto-fetch actual results from `/api/results-alt`
2. `reevaluateResults()` — compute probabilities
3. `runSimulation()` — build bracket
4. `applyLang()` — set language

Previously, simulation ran before auto-fetch completed, causing stale data.

### 9. What-If Simulation History
**File:** `compute-server.js` — `runMonteCarlo()`

Now returns `simulationHistory` in the response:
```javascript
return { ...mcResults, simulationHistory: history };
```

Frontend stores it: `WC26.simulationHistory = data.simulationHistory || []`

### 10. Dynamic Elo Display
**File:** `compute-server.js` — `runSimulation()`

Returns `dynamicElo` in the simulation response:
```javascript
return { ..., dynamicElo: { ...WC26.dynamicElo } };
```

Frontend stores it: `if(currentResults.dynamicElo) WC26.dynamicElo = currentResults.dynamicElo`

## Known Issues

### 1. Third-Place Assignment Mismatch (MEDIUM)
When the simulation predicts different group standings than actual results, the bracket shows different third-place teams. Example:
- Simulation: Iran (3rd in G) → bracket slot
- Actual FIFA: Bosnia Herzegovina (3rd in B) → bracket slot

**Root cause:** The bracket construction uses simulation-predicted standings, not actual FIFA results. This is inherent to the model's prediction accuracy.

**Possible fix:** Use actual results for bracket construction when available (requires passing actual group standings to `buildKOBracket`).

### 2. R16 Cross-Pairing Dependency (LOW)
The R16 cross-pairing uses hardcoded bracket indices. If the bracket array order changes, the R16 pairing will break.

**Mitigation:** The bracket array order is now fixed to match FIFA schedule. Any changes require updating both the bracket array and the R16 pairing.

### 3. Date Sorting Edge Case (LOW)
The date sorting uses zero-padded comparison: `"07/01"` vs `"07/10"`. This works correctly but requires the padding function to be applied consistently.

### 4. Third-Place Eligibility Constraints (LOW)
The backtracking algorithm for third-place assignment uses `eligible` arrays that exclude groups based on R16 opponent proximity. These constraints may not cover all edge cases for unusual group stage outcomes.

**Mitigation:** Greedy fallback handles cases where backtracking fails.

### 5. Compute Server Single-Threaded (INFO)
The compute server runs on a single Node.js thread. Heavy Monte Carlo simulations (50,000+ iterations) may block other requests.

**Current mitigation:** Simulations run on-demand, not continuously.

## Model Pipeline

### Group Stage Prediction
```
1. getFormAdjustedLambdas(home, away, formMap, date, marketProbs)
   → [lambda_home, lambda_away]
2. getBlendedProbs(home, away, formMap, date, marketProbs)
   → {win, draw, loss} probabilities
   Steps: Dixon-Coles → GBDT blend (20%) → Temperature scaling (T=1.15)
```

### Knockout Match Prediction
```
1. getFormAdjustedLambdas(home, away, formMap, date, marketProbs)
   → [lambda_home, lambda_away]
2. detScore(home, away, lh, la)
   → {ga90, gb90, ga, gb, method, lh, la}
   Method: Math.round(lambda) for 90-min score
   If draw: Elo-based penalty winner
```

### Monte Carlo Simulation
```
1. For each iteration (N=50,000):
   a. Build group standings from actual/simulated results
   b. Determine 32 qualifying teams
   c. Build R32 bracket (buildKOBracket)
   d. Simulate all knockout matches (simKO — random Poisson)
   e. Record: champion, finalist, semi, quarter, R16
2. Aggregate: count/N = probability
```
